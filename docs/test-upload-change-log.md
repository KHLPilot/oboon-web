# test-upload 기능 조정 정리

1.
- 변경 기능: PDF 파일 선택 해제(X 버튼)
- 문제/배경: 잘못 선택한 PDF를 다시 선택하려면 전체 입력 흐름을 반복해야 했다.
- 구현 방법: 선택 파일 배지 우측에 제거 버튼을 추가하고, 클릭 시 파일 배열에서 해당 인덱스를 제거하도록 상태 업데이트 로직을 적용했다.
- 기대 효과: 파일 선택 실수를 빠르게 수정할 수 있어 업로드 재시도 비용이 줄어든다.

2.
- 변경 기능: 비교 결과 자동 로딩
- 문제/배경: 매번 “비교하기”를 눌러야 결과가 갱신되어 사용 흐름이 끊겼다.
- 구현 방법: 후보 ID/비교 조건 변경 시 `useEffect`로 비교 로직을 자동 실행하도록 구성했다.
- 기대 효과: 입력-확인 흐름이 자연스러워지고 누락 클릭으로 인한 오동작이 줄어든다.

3.
- 변경 기능: 비교 반영 완료 메시지 위치 이동
- 문제/배경: 완료 상태가 화면 하단에 있어 사용자가 반영 결과를 즉시 인지하기 어려웠다.
- 구현 방법: 비교 결과 표 액션 영역(“선택한 값 반영하기” 주변)으로 메시지 렌더링 위치를 옮겼다.
- 기대 효과: 반영 성공/실패를 사용자 액션 지점에서 즉시 확인할 수 있다.

4.
- 변경 기능: 분양 상태 한글 매핑
- 문제/배경: READY 같은 영문 상태가 운영 화면에서 직관성이 떨어졌다.
- 구현 방법: DB 저장값(영문 코드)과 UI 표기값(한글)을 분리해 화면에서는 한글로 렌더링하도록 매핑 함수를 적용했다.
- 기대 효과: 비전문 사용자도 상태 의미를 즉시 이해할 수 있다.

5.
- 변경 기능: 추출 이미지 UI 공용 스타일 토큰 정리
- 문제/배경: 일부 카드/배지가 글로벌 토큰과 공용 UI를 사용하지 않아 화면 일관성이 깨졌다.
- 구현 방법: 공통 `Button`, `DropdownMenu`, 색상/타이포/보더 토큰 기반 클래스로 교체했다.
- 기대 효과: 테마 일관성과 유지보수성이 개선된다.

6.
- 변경 기능: 추출 이미지 카드 레이아웃 재구성
- 문제/배경: 이미지가 작거나 정보 배치가 분산되어 사용성이 낮았다.
- 구현 방법: 데스크탑 2열 카드, 카드 하단 드롭다운, 16:9 비율, `object-fit: contain`을 적용했다.
- 기대 효과: 이미지 가독성과 용도 지정 작업 속도가 향상된다.

7.
- 변경 기능: 용도 배지(현장사진/추가 현장사진) 노출
- 문제/배경: 각 이미지의 용도 상태를 빠르게 구분하기 어려웠다.
- 구현 방법: 이미지 좌상단 오버레이 배지로 상태를 표시하고 safe 톤 색상 토큰을 사용했다.
- 기대 효과: 한눈에 분류 상태를 파악해 오분류를 줄일 수 있다.

8.
- 변경 기능: 주택형(타입) 행 추가
- 문제/배경: PDF 추출 결과에 없는 타입을 수동으로 추가할 경로가 부족했다.
- 구현 방법: 타입 섹션에 “행 추가” 버튼을 두고 `result.unit_types` 배열에 기본 행을 append하도록 구현했다.
- 기대 효과: 신규 타입을 즉시 입력해 저장 파이프라인에 포함할 수 있다.

9.
- 변경 기능: 분양가 단위 보정(만원→원)
- 문제/배경: UI 입력 단위와 DB 저장 단위가 달라 값이 잘못 들어갈 수 있었다.
- 구현 방법: 저장 시 `toWonFromManwon` 변환을 적용해 DB에는 원 단위로 저장되게 했다.
- 기대 효과: 가격 데이터 정합성이 보장된다.

10.
- 변경 기능: 자동 저장 + 수동 저장 병행
- 문제/배경: 자동 저장만으로는 사용자가 반영 시점을 제어하기 어려웠다.
- 구현 방법: 용도/타입 변경 시 자동 저장 트리거를 유지하고, 별도 저장 버튼으로 명시 저장 경로를 추가했다.
- 기대 효과: 실시간 반영성과 사용자 제어성을 동시에 확보한다.

11.
- 변경 기능: 저장 컨텍스트 분기 강화
- 문제/배경: 기존 현장 업데이트 상황에서도 새 현장으로 저장되는 케이스가 발생했다.
- 구현 방법: `existingSnapshot`, `createdPropertyId` 상태를 기준으로 저장 경로를 명확히 분기하는 `handleSaveByContext`를 적용했다.
- 기대 효과: 저장 대상 혼선이 줄어들고 의도한 현장에만 반영된다.

12.
- 변경 기능: 기존 현장 저장 반영 범위 확대
- 문제/배경: 기존 저장 경로가 일부 섹션만 반영하여 실제 기대와 차이가 있었다.
- 구현 방법: properties/location/specs/timeline + 추출 이미지 동기화까지 한 번에 처리하도록 저장 흐름을 확장했다.
- 기대 효과: “기존 현장 업데이트”의 결과가 사용자 기대와 일치한다.

13.
- 변경 기능: 기존 현장 저장 시 `property_unit_types` 동기화 추가(핵심)
- 문제/배경: 신규 평면 타입을 추가해도 DB에 저장되지 않는 핵심 버그가 있었다.
- 구현 방법: `syncUnitTypesForProperty()`를 추가해 타입명 키 매칭 기준으로 기존 행은 update, 미존재 행은 insert 하도록 구현했다.
- 기대 효과: 신규 타입이 누락 없이 DB에 반영된다.

14.
- 변경 기능: 평면도 매핑 id 중심 처리
- 문제/배경: 문자열 타입명 매칭은 OCR/표기 변형에 취약했다.
- 구현 방법: 평면도 저장/삭제 시 `unitTypeId`를 우선 사용해 대상 행을 식별하고 갱신하도록 로직을 고정했다.
- 기대 효과: 타입명 변형에도 안정적으로 동일 행을 업데이트할 수 있다.

15.
- 변경 기능: 추출 이미지 해시 중복 방지
- 문제/배경: 동일 이미지 재업로드로 갤러리 제한 초과와 중복 데이터가 발생했다.
- 구현 방법: 이미지 해시 기반으로 기존 저장 이력을 조회해 동일 항목은 업로드를 건너뛰는 로직을 적용했다.
- 기대 효과: 업로드 실패(최대 장수 초과)와 중복 저장이 크게 감소한다.

16.
- 변경 기능: “사용 안함” 시 DB 정리 반영
- 문제/배경: 화면에서 비활성화해도 DB에 이전 이미지 연결이 남는 문제가 있었다.
- 구현 방법: 저장된 추출 이미지 맵을 기준으로 목적지 변경/비활성 항목은 gallery/main/floor_plan 연결을 삭제 동기화했다.
- 기대 효과: 화면 상태와 DB 상태가 일치한다.

17.
- 변경 기능: 갤러리 API `caption` 지원
- 문제/배경: 업로드된 이미지의 출처/역할 추적이 어려웠다.
- 구현 방법: `/app/api/property/gallery/route.ts` POST에서 `caption` 저장을 지원하도록 확장했다.
- 기대 효과: 후속 동기화와 중복 판단 시 식별 정확도가 올라간다.

18.
- 변경 기능: 드롭다운 장문 목록 스크롤
- 문제/배경: 옵션이 많아지면 드롭다운이 과도하게 길어져 페이지 사용성이 떨어졌다.
- 구현 방법: `/components/ui/DropdownMenu.tsx`에 최대 높이와 `overflow-y` 스크롤을 적용하고 표시 개수를 5개 수준으로 제한했다.
- 기대 효과: 긴 목록에서도 안정적인 UI 높이를 유지한다.

19.
- 변경 기능: 페이지 가로 스크롤 완화
- 문제/배경: 긴 에러 메시지/컨텐츠로 인해 좌우 스크롤이 발생했다.
- 구현 방법: `/app/layout.tsx`, `/app/globals.css`에 `overflow-x` 제어를 추가하고 긴 텍스트에는 `break-all` 처리를 적용했다.
- 기대 효과: 레이아웃이 화면 폭을 벗어나지 않고 안정적으로 유지된다.

20.
- 변경 기능: 타입 테이블 편집 중 폭 튐 현상 개선
- 문제/배경: 셀 편집 시 입력 요소가 폭을 밀어 표 레이아웃이 흔들렸다.
- 구현 방법: `EditableText`의 표시/버튼/입력 모드에 `min-w-0`, `max-w-full`, `overflow-hidden`을 적용하고, 표시/버튼 모드는 `w-fit`로 조정했다.
- 기대 효과: 편집 시 과도한 너비 변형 없이 기존 폭을 유지한다.

21.
- 변경 기능: 저장 완료 피드백 강화
- 문제/배경: 저장 성공 여부와 반영 범위를 사용자에게 즉시 전달하기 어려웠다.
- 구현 방법: `showAlert` 모달을 사용해 성공/실패를 즉시 알리고, 상태 메시지에 업로드/중복/삭제/동기화 카운트를 포함했다.
- 기대 효과: 작업 결과를 빠르게 검증하고 재시도 여부를 판단할 수 있다.

22.
- 변경 기능: 상세 페이지 평면도 다중 이미지 표시 보강
- 문제/배경: 평면도 다중 업로드가 되어도 상세 화면의 탐색 버튼 노출이 일관되지 않았다.
- 구현 방법: `/features/offerings/components/detail/offeringTypesAccordion.client.tsx`에서 다중 이미지 조건 및 이전/다음 버튼 표시 로직을 보강했다.
- 기대 효과: 타입별 다중 평면도 탐색이 명확해진다.

## 검증 체크리스트(수동 테스트 10개)

1. 새 PDF 선택 후 파일명 배지 X 클릭 시 해당 파일만 정확히 제거되는지 확인.
2. 유사 현장 선택 변경 시 비교 결과가 버튼 클릭 없이 자동 갱신되는지 확인.
3. 비교 반영 완료 메시지가 비교 표 액션 근처에 노출되는지 확인.
4. 상태값 READY 등 영문 코드가 화면에서 한글로 표시되는지 확인.
5. 타입 섹션에서 행 추가 후 입력값 저장 시 DB `property_unit_types`에 신규 행이 생성되는지 확인.
6. 기존 현장 업데이트 저장 시 properties/specs/timeline/location/unit_types가 모두 반영되는지 확인.
7. 동일 추출 이미지를 다시 저장할 때 업로드가 스킵되고 중복 카운트가 증가하는지 확인.
8. 추출 이미지 용도를 “사용 안함”으로 변경 후 저장하면 관련 DB 연결이 제거되는지 확인.
9. 드롭다운 옵션이 5개 초과일 때 높이가 고정되고 내부 스크롤이 동작하는지 확인.
10. 타입 테이블 셀 편집 중 페이지/표 폭이 갑자기 늘어나지 않는지 확인.
