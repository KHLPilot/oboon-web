## 주요 변경사항

### 1) 모바일 예약 CTA 문구 개선
- 대상 파일: `features/offerings/components/detail/OfferingDetailRight.tsx`
- 변경 내용:
  - 차단 계정 버튼 문구를 `예약 불가 계정`에서 `일반 회원만 예약 가능`으로 변경
  - 모바일에서 차단 계정일 때 보조 안내 문구 노출 추가:
    - `관리자/상담사 계정은 예약할 수 없습니다.`
  - 모바일에서 예약 가능한 상담사가 없을 때 문구를 간결화:
    - `상담 가능 상담사 없음`

### 2) 상담사 프로필 설정 화면 미리보기 기능 추가
- 대상 파일: `features/profile/components/ProfilePage.client.tsx`
- 변경 내용:
  - 상담사 프로필 편집 영역 액션에 `미리보기` 버튼 추가
  - 미리보기 모달 추가 (예약 모달의 상담사 프로필 보기 UI와 동일)
  - 추가 사진 클릭 시 확대 모달/이전/다음 이동 지원
  - 미리보기 데이터는 현재 입력 상태(프로필 이미지, 이름, 소개, 추가 사진)를 즉시 반영

### 3) 새 현장 등록 시 중복 현장명 사전 경고 강화
- 대상 파일: `app/company/properties/new/page.tsx`
- 변경 내용:
  - 현장명 입력 중(디바운스) 자동완성형 유사 현장 제안 카드 노출
  - 정확 일치 시: `혹시 이 현장인가요?`
  - 부분 일치 시: `이런 현장이 있어요`
  - 공백 변형(예: `디에이치클래스트`/`디에이치 클래스트`)도 정규화 비교로 감지
  - 한글 IME 조합 중에는 조회를 멈추고, 글자 완성 시점부터만 조회
  - 카드에서 `소속 신청` 버튼으로 즉시 소속 처리 가능
  - 소속 신청 성공 시 `마이페이지 > 소속 등록` 섹션으로 즉시 이동:
    - `/agent/profile#affiliation-section`
  - 정확 일치 중복일 때는 신규 등록 제출을 차단하고 기존 현장 소속 흐름으로 유도

## 사용자 영향
- 모바일 예약 화면에서 차단 사유를 더 직관적으로 이해 가능
- 상담사는 프로필 저장 전에 실제 노출 형태를 확인 가능
- 상담사가 현장명을 일부만 입력해도 기존 현장을 빠르게 찾아 바로 소속 처리 가능

## 배포 전 최종 확인 권장
- 모바일(`/offerings/[id]`)에서 아래 케이스 QA
  - 일반 회원 로그인: `상담 신청` 버튼 정상 동작
  - 상담사/관리자 로그인: `일반 회원만 예약 가능` + 보조 안내 문구 노출
  - 예약 가능 상담사 없음: `상담 가능 상담사 없음` 노출
- 상담사 마이페이지(`/agent/profile`)
  - `미리보기` 버튼 노출
  - 모달 UI가 예약 모달의 상담사 프로필 보기와 동일한지 확인
  - 추가 사진 확대/좌우 이동/닫기 동작 확인
- 새 현장 등록(`/company/properties/new`)
  - 현장명 일부 입력 시 유사 현장 제안 카드가 실시간 노출되는지 확인
  - 한글 조합 입력 중에는 조회가 발생하지 않고 글자 완성 후 조회되는지 확인
  - 카드에서 `소속 신청` 성공 후 `/agent/profile#affiliation-section`으로 이동되는지 확인
  - 정확 일치 중복 이름으로 `등록하기` 시 신규 등록이 차단되는지 확인
