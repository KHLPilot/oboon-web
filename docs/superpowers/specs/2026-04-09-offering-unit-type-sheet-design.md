# 분양가표 → 행 + 시트/패널 구조 재설계

작성일: 2026-04-09

## 목적

타입이 많을 때 아코디언 펼침 방식은 스크롤 공간을 크게 차지하고, 모바일에서 펼침/닫힘 인터랙션이 불편하다. 각 타입 행을 최소 비교 정보만 담은 압축 행으로 만들고, 상세 정보는 클릭 시 Bottom Sheet(모바일) / 우측 슬라이드 패널(데스크톱)로 열리도록 구조를 바꾼다.

## 문제 정의

- 타입이 5개 이상이면 아코디언이 화면 대부분을 차지해 타입 비교가 어렵다.
- 모바일에서 아코디언 펼침 후 스크롤이 끊기고 위치 파악이 어렵다.
- 접힌 상태에서도 조건 결과를 보여줘야 해서 행 높이가 줄어들지 않는다.

## 목표

- 접힌 행은 타입별 가격 비교에만 집중한 최소 정보로 구성한다.
- 행 클릭 시 모바일은 Bottom Sheet, 데스크톱은 우측 슬라이드 패널로 상세를 연다.
- 조건 검증 결과(배지 + 핵심 수치)는 접힌 행에도 표시해 열지 않아도 비교가 가능하다.

## 비목표

- 라우팅 변경 없음 (`/offerings/[id]/units/[unitId]` 상세 페이지 추가 안 함).
- 데이터 계산 로직(`offeringPriceTableLayout.ts`) 변경 없음.
- 기존 평면도 확대 `ImageModal` 교체 없음.

## 접힌 행 구조

각 타입 행은 클릭 가능한 단일 카드로 동작한다.

```
┌───────────────────────────────────────┐
│ 84A타입                           →  │
│ 16억 5,680                            │
│ 전용 59.9㎡ · 3룸 · 2욕실 · 84세대  │
├───────────────────────────────────────┤
│ ● 우선 검토  초기 1.4억 · 월 32.1%  │  ← 검증 후
│  or  [내 조건으로 확인]              │  ← 검증 전
└───────────────────────────────────────┘
```

- 타입명: `ob-typo-h4`
- 분양가 범위: `ob-typo-h2`, 시각적으로 가장 크게
- 보조 스펙: `ob-typo-caption`, muted, `buildOfferingUnitSpecSummary` 사용
- 조건 상태 행: `buildOfferingUnitConditionState` 사용, 기존 배지 색상 유지
- 행 전체가 하나의 `<button>` 또는 클릭 핸들러로 처리
- ChevronDown 제거 → ChevronRight(또는 `→`) 아이콘으로 교체

## 패널/시트 구조

### 모바일 (Bottom Sheet, `< lg`)

- 화면 하단에서 위로 슬라이드인 (`transform: translateY`)
- 높이: 최대 `85dvh`, 내부 스크롤 가능
- 드래그 핸들 표시 (상단 중앙 pill)
- 배경 반투명 오버레이 클릭 시 닫힘
- `body` 스크롤 잠금 (기존 `ImageModal`과 동일 방식)

### 데스크톱 (우측 슬라이드 패널, `≥ lg`)

- `position: fixed`, 우측에서 슬라이드인 (`transform: translateX`)
- 너비: `420px`
- 높이: `100dvh`, 내부 스크롤 가능
- 배경 반투명 오버레이 (`bg-black/40`) 클릭 시 닫힘
- 좌측 타입 목록 레이아웃은 변경 없음 (패널이 위에 겹침)

### 시트/패널 내부 콘텐츠 순서

1. **헤더**: 타입명 + 닫기 버튼
2. **분양가 정보**: 분양가 범위, 가격 공개 여부
3. **내 조건 기준** (검증 완료 사용자만): 계약금, 초기 필요 자금, 권장 현금, 예상 대출, 월 부담률
4. **판정 이유** (검증 완료 사용자만): 카테고리별 상태 칩 + 한 줄 설명
5. **타입 정보**: 전용/공급면적(㎡/평 토글), 방/욕실, 구조, 향, 세대수
6. **평면도**: 썸네일 클릭 → 기존 `ImageModal` 오버레이

## 컴포넌트 구조

```
offeringTypesAccordion.client.tsx   ← 아코디언 펼침 제거, 행 클릭 → sheet 열기
UnitTypeDetailSheet.tsx             ← 신규 (모바일 sheet + 데스크톱 패널 통합)
  └─ ImageModal                     ← 기존 재사용 (평면도 확대)
offeringPriceTableLayout.ts         ← 변경 없음
```

`UnitTypeDetailSheet`는 `open`, `unit`, `validation`, `onClose` props를 받는다. 내부에서 `window.matchMedia('(min-width: 1024px)')` 또는 Tailwind breakpoint 기반으로 모바일/데스크톱 렌더를 분기한다.

## 인터랙션 규칙

- 시트/패널이 열린 상태에서 다른 행 클릭 → 해당 타입으로 내용 교체 (닫고 다시 열지 않음)
- ESC 키로 닫힘
- 평면도 썸네일 클릭 → `ImageModal` 열림 (시트/패널 위 레이어)
- 검증 전 CTA(`내 조건으로 확인`) → 시트 닫기 + `#condition-validation` 스크롤

## 접근성

- 시트/패널: `role="dialog"`, `aria-modal="true"`, `aria-label="타입명 상세"`
- 닫기 버튼: `aria-label="닫기"`
- 행 버튼: `aria-haspopup="dialog"`, `aria-expanded` 제거 (더 이상 아코디언 아님)

## 완료 기준

- [ ] 접힌 행이 타입명 + 가격 + 보조스펙 + 조건 상태만 표시한다.
- [ ] 행 클릭 시 모바일은 Bottom Sheet, 데스크톱은 우측 패널이 열린다.
- [ ] 시트/패널 내부에서 분양가 → 조건 기준 → 판정 이유 → 타입 정보 → 평면도 순서로 표시된다.
- [ ] 평면도 클릭 시 기존 `ImageModal`이 시트/패널 위에 열린다.
- [ ] 다른 행 클릭 시 시트/패널 내용이 교체된다.
- [ ] ESC 및 배경 클릭으로 닫힌다.
- [ ] `body` 스크롤이 시트/패널 열림 상태에서 잠긴다.
