# 맞춤 현장 평형 패널 설계

작성일: 2026-04-07

## 목적

현재 맞춤 현장 추천 API는 평형별 평가 결과를 내려주지만, 추천 리스트 카드에서는 이를 사용하지 않는다. 그 결과 사용자는 "이 현장이 나와 맞는가"는 빠르게 볼 수 있어도, "이 현장 안에서 어떤 평형이 나와 맞는가"는 리스트 단계에서 확인할 수 없다.

이번 설계의 목적은 추천 리스트의 가벼운 탐색 경험은 유지하면서, 평형별 상세 정보를 리스트 단계에서 자연스럽게 열어볼 수 있도록 정보 구조를 재설계하는 것이다.

## 문제 정의

- 현재 추천 리스트 카드는 현장 단위 요약만 보여준다.
- 평형별 데이터는 상세 페이지의 조건 검증 카드 안에서만 노출된다.
- 추천 API는 `best_unit_type`, `unit_type_results`를 내려주지만 추천 리스트 훅과 카드가 이 데이터를 사용하지 않는다.
- 카드 안에 평형 정보를 직접 많이 넣으면 리스트 비교성이 무너지고, 카드가 과도하게 무거워진다.

## 목표

- 리스트 단계에서 평형별 추천 결과를 확인할 수 있어야 한다.
- 카드 자체는 여전히 현장 요약과 상세 페이지 진입 역할을 유지해야 한다.
- 평형별 정보는 "의도적으로 여는 상세 레이어"로 분리해야 한다.
- 데스크톱과 모바일에서 같은 정보를 제공하되, 화면 제약에 맞게 다른 표현을 사용해야 한다.

## 비목표

- 맞춤 현장 카드 자체를 평형 비교 카드로 전환하지 않는다.
- 리스트 카드 안에 모든 평형 정보를 항상 노출하지 않는다.
- 현장 상세 페이지의 타입별 확인 UI를 이번 작업에서 대체하지 않는다.

## 최종 방향

추천 리스트 카드는 현장 요약 레이어로 유지한다. 평형별 정보는 카드 내부의 별도 버튼으로 여는 상세 레이어로 분리한다.

- 카드 본문 클릭: 기존처럼 현장 상세 페이지로 이동
- 별도 버튼 클릭: 평형별 상세 정보 열기
- 동시에 하나의 현장만 평형 상세를 열 수 있음
- 데스크톱: 선택된 카드 아래에 인라인 확장 패널 표시
- 모바일: 카드 아래 확장 대신 전용 바텀시트 표시

이 방향은 "리스트는 빠르게 훑고, 필요할 때만 평형을 파고드는 구조"를 유지한다.

## 카드 구조

추천 카드의 구조는 아래 네 층으로 정리한다.

1. 현장 요약
현장명, 지역/상태, 매칭률, 가격 등 현재 카드의 핵심 요약 정보를 유지한다.

2. 핵심 수치
현장 단위 가격/부담률/매칭 등급 등 지금 카드가 제공하는 빠른 판단용 정보는 유지한다.

3. 대표 평형 미리보기
버튼 위에 한 줄짜리 미리보기를 추가한다.
예시:

- `추천 평형 84㎡ A, 59㎡ B 외 2개`
- `추천 평형 59㎡ 단일`

대표 평형은 추천 순 상위 평형 기준으로 노출한다.

4. 평형 상세 진입 버튼
버튼 문구는 명시적으로 작성한다.
예시:

- `평형별 자세히 보기`
- `추천 평형 보기`

버튼은 카드 클릭 동작과 분리되어야 하며, 카드 전체 클릭과 의미가 겹치지 않아야 한다.

## 평형 정렬 기준

평형 리스트는 추천 순으로 정렬한다.

정렬 우선순위:

1. 평형별 총점 내림차순
2. 월 부담률 오름차순
3. 분양가 오름차순

사용자 입장에서는 "가장 나에게 맞는 평형"을 먼저 보는 것이 자연스럽다. 가격 순이나 면적 순보다 의사결정 흐름에 더 맞다.

## 데스크톱 상호작용

데스크톱에서는 카드 하단에 인라인 패널을 연다.

- 사용자가 카드의 평형 버튼을 누르면 해당 카드 아래에 패널이 열린다.
- 이미 다른 카드의 패널이 열려 있다면 닫고, 새 카드의 패널만 연다.
- 같은 카드의 버튼을 다시 누르면 패널을 닫는다.
- 리스트에서 선택된 카드와 열린 평형 패널의 상태는 동일한 기준으로 관리한다.

패널은 현재 카드의 시각적 폭과 정렬을 따라가야 하며, 별도의 모달로 분리하지 않는다. 사용자는 리스트 문맥 안에서 현장 간 비교를 계속할 수 있어야 한다.

## 모바일 상호작용

모바일에서는 평형 버튼 클릭 시 바텀시트를 연다.

- 카드 아래 인라인 확장은 사용하지 않는다.
- 바텀시트 상단에는 현장명, 대표 등급, 현장 요약을 짧게 고정한다.
- 바텀시트 본문에는 평형 리스트를 스크롤 가능한 형태로 제공한다.
- 닫기 버튼과 스와이프/배경 탭 닫기를 지원할 수 있다.

모바일에서 인라인 확장을 쓰지 않는 이유는 상세형 데이터가 카드 높이를 지나치게 키워 스크롤 맥락을 쉽게 깨뜨리기 때문이다.

## 평형 패널 정보 구조

평형 패널은 "추천 순 평형 리스트"라는 명확한 제목을 가진다.

각 평형은 처음에는 요약 상태로 렌더링한다. 사용자가 개별 평형 행을 누르면 세부 정보가 열리는 2단 구조를 사용한다.

### 요약 상태에 표시할 정보

- 평형명
- 전용면적
- 분양가
- 매칭 등급
- 매칭률
- 월 부담률

### 확장 상태에 추가로 표시할 정보

- 한 줄 요약 코멘트
- 카테고리 점수
  - 자금력
  - 소득
  - LTV·DSR
  - 주택 보유
  - 구매 목적
  - 시점

즉, 평형 패널 전체는 상세형 정보 구성을 따르되, 개별 평형 카드 내부는 요약 -> 확장 2단계로 설계한다.

## 카테고리 점수 표현

카테고리 점수는 숫자 나열보다 비교 가능한 시각 요소로 표현한다.

권장 표현:

- 카테고리명
- 등급 칩 또는 작은 색상 배지
- 필요 시 점수값

표현 예:

- `자금력 GREEN`
- `소득 LIME`
- `LTV·DSR YELLOW`

데스크톱/모바일 모두 과도한 표 형태보다 칩/미니 바 형태가 적합하다. 목적은 정밀 계산 표시가 아니라 "어느 항목이 강점/약점인지"를 빠르게 읽게 하는 것이다.

## 대표 평형 미리보기 규칙

카드 미리보기 한 줄은 아래 원칙을 따른다.

- 추천 순 상위 1~2개 평형명만 노출
- 나머지는 `외 n개`로 축약
- 평형이 1개뿐이면 `단일` 또는 단일 평형명으로 표시

예시:

- `추천 평형 84㎡ A, 59㎡ B 외 2개`
- `추천 평형 74㎡ A 외 1개`
- `추천 평형 59㎡ 단일`

목적은 사용자가 버튼을 누르기 전에 "무엇이 열리는지"를 이해하게 하는 것이다.

## 상태 관리 원칙

추천 리스트는 한 번에 하나의 평형 상세만 열 수 있다.

필요한 상태는 아래 두 종류다.

- 데스크톱용 열린 평형 패널 대상 현장 ID
- 모바일용 열린 바텀시트 대상 현장 ID 또는 아이템

기존 선택 상태와 분리할지 통합할지는 구현 단계에서 결정할 수 있지만, 사용자 경험 기준은 명확해야 한다.

- 카드 선택과 평형 상세 열림은 같은 대상 현장을 가리켜야 한다.
- 카드 선택만으로 자동으로 평형 패널이 열리면 안 된다.
- 반드시 버튼 클릭으로만 평형 상세가 열려야 한다.

## 데이터 사용 원칙

현재 추천 API가 내려주는 `unit_type_results`, `best_unit_type`를 리스트 단계에서 활용한다.

필요한 확장:

- 추천 훅의 응답 타입에 평형 관련 필드 추가
- 추천 카드 및 모바일 시트에서 사용할 평형 뷰모델 구성
- 대표 평형 미리보기 문자열 계산 유틸 추가

이번 설계에서는 API를 새로 바꾸기보다, 이미 내려오는 필드를 프론트가 실제로 사용하도록 연결하는 쪽을 우선한다.

## 접근성 및 상호작용 규칙

- 카드 본문 클릭과 평형 버튼 클릭의 역할은 명확히 분리한다.
- 평형 버튼은 버튼 역할과 라벨을 가진다.
- 데스크톱 패널과 모바일 시트 모두 키보드 포커스 진입이 가능해야 한다.
- 확장/축소 상태는 스크린리더에 전달되어야 한다.
- 개별 평형 행도 버튼 또는 아코디언 패턴으로 접근 가능해야 한다.

## 예외 처리

### 평형 결과가 없는 경우

- 버튼을 숨기거나 비활성화할 수 있다.
- 카드 미리보기 문구도 노출하지 않는다.

### 평형이 1개뿐인 경우

- 미리보기 문구는 단일 평형 기준으로 단순화한다.
- 패널 안에서도 아코디언 없이 기본 확장 상태로 렌더링하는 것을 고려할 수 있다.

### 비로그인/마스킹 상태

- 평형 패널에서 비공개 수치가 있는 경우 동일한 마스킹 원칙을 적용한다.
- 다만 평형 자체의 존재와 이름/전용면적 등 공개 가능한 정보는 유지할 수 있다.

## 테스트 관점

핵심 테스트 항목은 아래와 같다.

- 추천 응답의 평형 데이터가 훅에서 정상 파싱되는지
- 대표 평형 미리보기 문자열이 평형 수별로 올바르게 생성되는지
- 버튼 클릭 시 카드 본문 이동과 충돌하지 않는지
- 데스크톱에서 단일 패널만 열리는지
- 모바일에서 버튼 클릭 시 바텀시트가 열리는지
- 개별 평형 확장/축소가 정상 동작하는지
- 평형 정보가 없을 때 버튼/미리보기가 숨겨지는지

## 구현 범위

이번 구현은 아래 범위로 제한한다.

- 맞춤 현장 추천 훅의 평형 데이터 수용
- 추천 카드 하단의 대표 평형 미리보기와 별도 버튼 추가
- 데스크톱 인라인 평형 패널 추가
- 모바일 평형 바텀시트 추가
- 평형별 요약/확장 UI 추가

이번 범위에 포함하지 않는 것:

- 상세 페이지 조건 검증 카드의 구조 개편
- 추천 API 정렬/평가 로직 자체의 변경
- 평형 비교를 위한 별도 전용 페이지 신설

## 결정 사항 요약

- 평형 상세는 카드 본문이 아니라 별도 버튼으로 연다.
- 추천 리스트에서는 한 번에 하나의 현장만 평형 상세를 연다.
- 평형 정렬 기준은 추천 순이다.
- 평형 정보는 상세형으로 보여주되, 개별 평형은 2단 확장 구조를 쓴다.
- 데스크톱은 카드 아래 인라인 패널, 모바일은 바텀시트를 사용한다.
- 카드에는 대표 평형 미리보기 한 줄을 추가한다.
