## 1. 서비스 비전

### 1-1. 서비스 정의

> OBOON(오늘의 분양)은 분양 현장 탐색부터 조건 검증, 상담사 연결, 방문 인증까지
> **분양 구매 의사결정의 전 과정을 하나의 플로우로 연결**하는 플랫폼이다.

### 1-2. 문제 정의

| 문제 | 현황 |
|------|------|
| 정보 비대칭 | 분양 현장 정보가 분산되어 있어 고객이 직접 비교하기 어렵다 |
| 조건 불확실성 | 내 상황에 이 분양이 맞는지 판단할 기준이 없다 |
| 상담 접근성 | 상담사 연결이 불투명하고 신뢰하기 어렵다 |
| 방문 비효율 | 현장 방문 후 후속 프로세스가 연결되지 않는다 |

### 1-3. 핵심 가치 제안

```
고객: 내 조건에 맞는 분양 현장을 찾고, 믿을 수 있는 상담사와 연결된다
상담사: 검증된 고객과 효율적으로 상담한다
운영사: 분양 데이터와 상담 흐름을 플랫폼에서 관리한다
```

## 2. 사용자 페르소나

### P1. 실수요 고객 (핵심)

- **프로필**: 30-40대, 내 집 마련 또는 갈아타기를 고민 중
- **목표**: 지금 내 상황에서 살 수 있는 분양인지 확인하고 싶다
- **불안**: 대출이 될지 모르겠다, 상담사를 믿을 수 있는지 모르겠다
- **행동 패턴**: 네이버 카페/부동산 앱 검색 → 현장 방문 → 계약 고민

### P2. 투자 목적 고객

- **프로필**: 40-50대, 수익형 자산 포트폴리오 관심
- **목표**: 전매제한·규제 여부를 빠르게 확인하고 수익성 판단
- **불안**: 규제 지역 리스크, 전매 제한 조건
- **행동 패턴**: 지역별 비교 탐색 → 조건 검증 → 빠른 의사결정

### P3. 분양대행사 상담사

- **프로필**: 분양대행사 소속, 여러 현장 동시 관리
- **목표**: 배정된 현장의 고객 예약을 효율적으로 관리
- **불안**: 무응답 고객, 스케줄 충돌, 방문 확인 번거로움
- **행동 패턴**: 앱 알림 → 예약 확인 → 채팅 응대 → 방문 확인

### P4. 시행/시공사 담당자

- **프로필**: 현장 분양 담당자
- **목표**: 현장 정보를 플랫폼에 등록하고 배정된 상담사를 관리
- **행동 패턴**: 현장 등록 → 평형/시설 입력 → 갤러리 업로드 → 상담사 배정

## 3. 핵심 사용자 여정

### 3-1. 고객 핵심 여정: 탐색 → 검증 → 예약 → 방문

```
[홈 진입]
    ↓
[현장 탐색] — 지역 필터 / 지도 뷰 / 목록 뷰
    ↓
[현장 상세] — 평형·위치·타임라인·갤러리 확인
    ↓
[조건 검증] — 현금/소득/보유주택/신용 입력 → GREEN/YELLOW/RED
    ↓
    ├─ GREEN → 방문 예약 바로 진행
    ├─ YELLOW → 사전 상담 권장 → 채팅 또는 예약
    └─ RED → 대안 현장 추천 + 상담 연결
    ↓
[예약 확정] — 상담사 슬롯 선택 → 상담 요청 → 상담사 확정
    ↓
[현장 방문] — GPS 인증 또는 수동 승인
    ↓
[후속] — 채팅 계속 / 커뮤니티 후기 작성
```

### 3-2. 상담사 여정: 수신 → 응대 → 확정 → 방문 확인

```
[알림 수신] — 신규 예약 요청
    ↓
[예약 확인] — 슬롯 확인 후 수락/거절
    ↓
[채팅 응대] — 고객과 사전 채팅
    ↓
[방문 당일] — 고객 도착 알림 수신 / 수동 승인
    ↓
[상담 완료] — 정산 처리
```

### 3-3. 회사 여정: 등록 → 관리 → 상담사 배정

```
[현장 등록] — PDF 업로드 or 직접 입력
    ↓
[정보 편집] — 위치, 사양, 평형, 갤러리
    ↓
[상담사 배정] — 현장별 담당 상담사 연결
    ↓
[현황 확인] — 방문/상담 통계
```

## 4. 기능별 기획 상세

### 4-1. 인증 & 온보딩

#### 회원가입 플로우

```
이메일 입력 → 비밀번호 설정 → 이메일 인증 → 닉네임/아바타 설정 → 온보딩
```

**정책**:
- 이메일 인증을 완료해야 서비스 이용 가능
- 인증 미완료 계정은 24시간 후 자동 삭제
- 탈퇴 후 30일 이내 재가입 시 계정 복구 유도

**소셜 로그인**:
- Google, Naver 지원
- 소셜 로그인 첫 진입 시 닉네임 설정 후 온보딩

#### 약관 동의

- 서비스 이용약관 (필수)
- 개인정보 처리방침 (필수)
- 마케팅 수신 동의 (선택)
- 동의 이력 DB 보관 (법적 요건)

### 4-3. 현장 상세

#### 조건검증 카드 노출 정책

| 상태 | 노출 내용 |
|------|-----------|
| 비로그인 | "로그인 후 내 조건을 확인하세요" CTA |
| 로그인 + 프로필 없음 | 조건 입력 폼 (인라인) |

- 현장에 배정된 상담사가 있을 때만 "예약하기" 버튼 표시
- 배정된 상담사 없으면 "상담 문의" 표시 (추후 연결)

### 4-4. 맞춤 조건 검증

#### 검증 카테고리별 의미

| 카테고리 | 의미 | 주요 지표 |
|----------|------|-----------|
| 현금(Cash) | 계약금+중도금 감당 가능 여부 | 최소현금, 권장현금 |
| 부담(Burden) | 월 상환 부담이 소득 대비 적정한지 | 월부담률 (권장 40% 이하) |
| 위험(Risk) | 규제·신용·목적 리스크 | 다주택·규제지역·전매제한 |

#### 등급별 UX 정책

| 등급 | 색상 | 메인 CTA | 부가 안내 |
|------|------|----------|----------|
| GREEN | 초록 | "방문 예약하기" | 예상 월납부액 표시 |
| YELLOW | 노랑 | "사전 상담 신청" | 주의 항목 1-2가지 강조 |
| RED | 빨강 | "대안 현장 보기" | 이유 명확히 표시, 상담 유도 |

#### 맞춤 현장 추천 기준

1. 조건 프로필로 평가 가능한 현장 중 GREEN 우선
2. YELLOW 현장 (단, 리스크 코드 명시)
3. 지역 기반 가중치 (최근 조회 지역)

### 4-5. 상담 & 채팅

#### 예약 가능 조건

- 상담사가 근무시간/슬롯을 설정한 경우에만 예약 가능
- 고객 1인이 동시에 동일 상담사에게 중복 예약 불가

#### 상담 상태 전환 정책

| 전환 | 주체 | 조건 |
|------|------|------|
| requested → confirmed | 상담사 | 슬롯 가용 확인 후 수락 |
| requested → rejected | 상담사 | 사유 없이 거절 가능 |
| confirmed → cancelled | 고객 또는 상담사 | 방문 24시간 전까지 |
| confirmed → completed | 시스템 | 방문 인증 완료 후 |

#### 채팅 정책

- 상담 확정(confirmed) 상태에서만 채팅 활성화
- 취소/거절된 상담의 채팅 내역은 읽기 전용 유지
- 파일/이미지 전송 미지원 (텍스트만)

### 4-6. 방문 인증

#### GPS 인증 정책

- 현장 좌표 반경 **300m 이내** 진입 시 자동 인증
- 위치 권한 미허용 시 수동 승인 요청 안내

#### 수동 승인 정책

- 고객이 요청 → 상담사가 앱에서 승인 (사진 등 별도 증빙 없음)
- 상담사 미응답 시 24시간 후 자동 승인 처리 (운영 정책)
- 승인/거절 시 고객에게 알림 발송

### 4-7. 커뮤니티

#### 게시글 작성 정책

| 항목 | 정책 |
|------|------|
| 현장 연결 | 게시글에 특정 현장 태깅 가능 (선택) |
| 카테고리 | visited(방문) / thinking(고민) / property_qna(현장Q&A) |

| 리포스트 | 타인 글을 내 피드에 공유 (원본 연결 유지) |

#### 커뮤니티 노출 정책

- `visited` 탭: 방문 인증이 완료된 현장에 대한 글만 분류
- `property_qna` 탭: 특정 현장에 질문 태그된 글. 현장 상세 페이지에서도 노출
- 삭제된 계정의 글: 작성자명 "탈퇴한 사용자"로 표시

## 5. 비즈니스 로직 정책

### 5-1. 상담사 승인 프로세스

```
가입 (agent_pending) → 관리자 검토 → 승인(agent) / 거절(user 유지)
```

- 승인 거절 시 일반 사용자로 유지
- 승인 완료 시 상담사 전용 기능 활성화 + 알림 발송

### 5-2. 현장 가격 공개 정책

- `is_price_public = true`: 목록/상세에 가격 노출
- `is_price_public = false`: "가격 문의" 표시, 조건검증 메트릭 마스킹
- 평형 단위로 공개 여부 설정 가능

### 5-3. 정산 정책

- 상담 완료(방문 인증) 후 정산 대상 생성
- 환불: 상담 취소 시 처리 (`/api/consultations/[id]/refund`)
- 리워드: 상담사에게 지급 (`/api/consultations/[id]/reward-payout`)
- 관리자 정산 현황 대시보드에서 확인

### 5-4. 조건 프로필 갱신 정책

- 고객이 조건 입력 시 `condition_validation_profiles` 테이블에 upsert
- Cron으로 주기적 재평가 (현장 가격 변동 반영)
- 맞춤 추천 결과는 프로필 갱신 시 자동 업데이트

### 5-5. POI 갱신 정책

- 현장 등록 후 관리자가 수동 트리거 또는 Cron 자동 실행
- 카카오 API 기반 주변 시설 크롤링
- 카테고리: HOSPITAL / CLINIC_DAILY / MART / SUBWAY / HIGH_SPEED_RAIL / SCHOOL / DEPARTMENT_STORE / SHOPPING_MALL

## 6. UX 원칙

### 6-1. 정보 계층

```
탐색 (목록/지도) → 검증 (조건 맞는지) → 연결 (상담사) → 행동 (방문)
```

- 각 단계가 명확한 다음 행동(CTA)을 제시한다
- 이전 단계의 컨텍스트(어떤 현장, 어떤 등급)가 다음 단계로 이어진다

### 6-2. 게이팅 정책

| 기능 | 비로그인 | 로그인 |
|------|----------|--------|
| 현장 목록 탐색 | 허용 | 허용 |
| 현장 상세 | 허용 | 허용 |
| 조건 검증 | 로그인 유도 | 허용 |
| 예약/상담 | 로그인 유도 | 허용 |
| 커뮤니티 읽기 | 허용 | 허용 |
| 커뮤니티 작성 | 로그인 유도 | 허용 |
| 브리핑 | 허용 | 허용 |

### 6-3. 빈 상태 (Empty State) 정책

- 현장이 없는 지역 탭: "해당 지역의 분양 현장이 없습니다"
- 알림 없음: "새 알림이 없습니다"
- 조건검증 미입력: 입력 폼 인라인 표시 (별도 페이지 이동 없음)

### 6-4. 모바일 우선

- 모든 화면 모바일 최적화 설계
- 지도 화면: 모바일에서 하단 시트 형식으로 목록 표시
- 채팅: 모바일 키보드 올라올 때 입력창 고정

## 7. 콘텐츠 전략

### 7-1. 카피라이팅 원칙

- 수치 강조: "예상 월납부 OO만원", "최소현금 OO억"
- 불안 해소형: "지금 당장 살 수 있는지 확인해보세요"
- 행동 유도형: "무료 상담 신청하기"

> 자세한 카피는 `docs/COPYWRITING.md` 참고

### 7-2. 브리핑 콘텐츠 운영 기준

- 주 2-3회 업데이트 목표
- 오분 오리지널: 시리즈 형식, 읽는 순서 있음
- 일반 브리핑: 외부 큐레이션 + 자체 작성 혼합
